## Хеширование - это
➢Хэширование - преобразование информации с помощью особых математических формул.
➢В результате хеширования возникает хэш -
отображение данных в виде короткой,
в идеале уникальной, строки для
каждого набора данных.![[Pasted image 20260112164815.png]]
### Хеш-функция - это
функция, осуществляющая преобразования массива входных данных произвольной длины в выходную битовую строку установленной длины, выполняемое определенным алгоритмом.

  H: K → {0, …, m-1}
➢K - множество ключей
➢Н(К) - Хэш-код ключа K
➢m - параметр Хэш-функции, количество ячеек Хэш-таблицы

➢«Хорошая» хэш-функция имеет следующие свойства:
○ Считается за линейное время
○ Имеет не очень большие значения ( до 64 бит)
○ Имеет равномерное распределение элементов, в случае коллизий (совпадения хэш-кода у двух разных элементов)

### Хеш-таблица - это 
➢Хэш-таблицы - структура данных, которая позволяет хранить пары (ключ, значение) и осуществлять доступ к элементу по ключу.
![[Pasted image 20260112165053.png]]
## Коллизии
➢Коллизия - совпадение Хэш-значений при разных ключах:
H(K_1) = H(K_2), при K_1 != K_2
➢Способы разрешения коллизий:
Методом цепочек
Методом открытой адресации (например, линейное исследование, квадратичное исследование и двойное хэширование)

### Метод цепочек
➢При совпадении хеш-значений элементы связываются в цепочку. Таким образом увеличивается место под
хэш-таблицу.
![[Pasted image 20260112165910.png]]
### Открытая адресация
➢В основе открытой адресации лежат следующие принципы
○Количество элементов должно быть меньше или равно размеру хэш-таблицы.
○При вставке ключа выполняется проверка ячейки (занята она или нет). Если свободна, то происходит запись, иначе происходит поиск альтернативного места.

*Линейное исследование*
Ячейки хеш-таблицы последовательно просматриваются с некоторым фиксированным интервалом k между ячейками (обычно k = 1).
Для того, чтобы все ячейки оказались просмотренными по одному разу, необходимо, чтобы k было [взаимно-простым](https://ru.wikipedia.org/wiki/%D0%92%D0%B7%D0%B0%D0%B8%D0%BC%D0%BD%D0%B0%D1%8F_%D0%BF%D1%80%D0%BE%D1%81%D1%82%D0%BE%D1%82%D0%B0) с размером хеш-таблицы.
![[Pasted image 20260112170047.png]]

*Квадратичное исследование*
Интервал между ячейками с каждым шагом увеличивается на константу. Если размер хеш-таблицы равен степени двойки, то одним из примеров последовательности, при которой каждый элемент будет просмотрен по одному разу, является hash(x) mod N, (hash(x) + 1) mod N, (hash(x) + 3) mod N, (hash(x) + 6) mod N (треугольные числа)![[Pasted image 20260112170156.png]]

*Двойное хеширование*
Интервал между ячейками фиксирован, как при линейном пробировании, но, в отличие от него, размер интервала вычисляется второй, вспомогательной хеш-функцией, а значит, может быть различным для разных ключей. Значения этой хеш-функции должны быть ненулевыми и взаимно-простыми с размером хеш-таблицы, что проще всего достичь, взяв [простое число](https://ru.wikipedia.org/wiki/%D0%9F%D1%80%D0%BE%D1%81%D1%82%D0%BE%D0%B5_%D1%87%D0%B8%D1%81%D0%BB%D0%BE) в качестве размера, и потребовав, чтобы вспомогательная хеш-функция возвращала значения от 1 до N — 1.


||В худшем случае|В лучшем случае|
|Поиск элемента|О(n)|O(1)|
|Удаление элемента|O(n)|O(1)|
|Вставка элемента|O(n)|O(1)|
