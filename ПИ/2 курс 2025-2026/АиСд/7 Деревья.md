## Дерево - это
➢Определение
  Дерево - это связный граф без циклов.
➢Свойства
○ n вершин
○ n-1 ребер
○ 1 путь для 2-х любых вершин
## Корневое дерево - это
Корневое дерево - дерево с отмеченной вершиной - корнем.
2 - корень
1, 3, 7 - дети 2
3 - родитель 4, 5
6, 5, 4, 7 - листы
![[Pasted image 20260112173121.png]]

## Определения
➢Высота дерева
длина самого дальнего пути от корня к листу
➢n-й уровень в дереве
множество узлов, находящихся в дереве на высоте n
➢Рекуррентное определение
дерево - это корень и несколько деревьев, которые являются детьми этого корня.
Примечание: дерево может быть пустым

### Способы представления дерева
*➢Массив родителей*
в ячейку с текущим номером записывается родителей соответствующей вершины, причем для корневой вершины записывается сам корень

*➢Списки детей*
в элемент массива записывается массив детей текущего узла

*➢Связный список с ссылками на детей и родителя*

*➢Бинарное дерево* - дерево, каждый узел которого имеет максимум 2 потомка.

### Обход дерева
#### Обходы в глубину (Depth-first traversal)
*Центрированый* 
➢Сначала посещается левое поддерево
➢Затем корень
➢В конце правое поддерево
![[Pasted image 20260112173608.png]]
Порядок посещения на примере: ABCDEFGHI

*Прямой*
➢Сначала посещается корень
➢Затем левое поддерево
➢В конце правое поддерево
![[Pasted image 20260112173813.png]]
Порядок посещения на примере: FBADCEGIH

*Обратный*
➢Сначала посещается левое поддерево
➢Затем правое поддерево
➢В конце - корень
![[Pasted image 20260112173852.png]]
Порядок посещения на примере: ACEDBHIGF*

*В ширину*
1.Используется очередь
2.В очередь добавляется корень
3.Корень берется из очереди и обрабатывается
4.В очередь добавляются все дети корня
5.Пункты 3 и 4 повторяются, пока очередь не станет пустой
![[Pasted image 20260112174039.png]]
Порядок посещения на примере: FBGADICEH

## Бинарное дерево поиска
➢Бинарное дерево, для которого выполняются следующие условия:
●оба поддерева — левое и правое — являются бинарными деревьями поиска;
●у всех узлов левого поддерева произвольного узла X значения ключей данных меньше, нежели значение ключа данных самого узла X;
●у всех узлов правого поддерева произвольного узла X значения ключей данных больше либо равны, нежели значение ключа данных самого узла X.

### Основные операции

*➢Поиск*
Рекурсивный обход по дереву, сравнивая значение элемента с текущим, чтобы определить, в какую сторону дальше продвинуться.
![[Pasted image 20260112174226.png]]


*➢Вставка*
Сначала производится поиск места, в которое данный узел будет корректно поместить, а затем уже создается новый узел.
![[Pasted image 20260112174252.png]]


*➢Удаление*
При удалении стоит обратить внимание на 3 допустимые ситуации:
-удаление листа
-удаление узла с 1 ребенком
-удаление узла с 2 детьми
